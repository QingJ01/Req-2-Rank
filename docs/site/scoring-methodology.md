# 评分方法

Req2Rank 会聚合多个评审模型输出，并给出包含可靠性信号的最终得分。

## 评分维度与权重

- 功能完整性：30%
- 代码质量：25%
- 逻辑正确性：25%
- 安全性：10%
- 工程实践：10%

## 可靠性信号

- 评审一致性（IJA）：由评审分数离散程度推导的一致性等级。
- 置信区间（CI95）：聚合得分的不确定性范围。

## 聚合流程

1. 收集所有评审模型的维度分数。
2. 计算一致性指标。
3. 按权重聚合维度分数。
4. 附加 CI95 与告警标记。

## 结果展示位置

- 本地 CLI 报告（`req2rank report <run-id>`）
- 导出文件（`req2rank export ...`）
- Hub 排行榜条目（得分 + CI + 验证状态）
